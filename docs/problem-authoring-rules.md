# Problem Authoring Rules & Skills

이 문서는 CodeDash에서 새로운 문제를 출제할 때 필요한 기준을 정의한다.

## 1) 필수 룰 (Must)

1. `lib/problems.ts`의 `Problem` 타입을 완전히 만족해야 한다.
2. `id`는 고유하고 URL-safe 문자열이어야 한다. (예: `two-sum-variation`)
3. `functionName`과 `starterCode`의 함수 시그니처는 반드시 일치해야 한다.
4. `testCases`의 `input`은 함수 인자 순서와 정확히 동일해야 한다.
5. 모든 `testCases.expected`는 JSON 직렬화 기준으로 비교 가능한 값이어야 한다.
6. `examples`의 `input/output`은 실제 `testCases`와 모순되면 안 된다.
7. `constraints`는 문제에서 허용하는 입력 범위를 빠짐없이 포함해야 한다.
8. 최소 1개 이상의 정상 케이스와 1개 이상의 경계 케이스를 포함해야 한다.

## 2) 권장 룰 (Should)

1. 테스트 케이스는 최소 3개 권장:
   - 기본 동작 1개
   - 경계값 1개
   - 함정(오답 유도) 1개
2. 난이도와 제약을 일관되게 맞춘다.
   - `Easy`: 구현/기초 자료구조
   - `Medium`: 패턴 응용, 복합 조건
   - `Hard`: 고급 최적화/복합 상태 관리
3. 설명은 “입력/요구사항/반환값” 순서로 짧고 명확하게 쓴다.
4. 카테고리는 기존 분류 체계를 우선 사용한다. (Hash, Stack, DP 등)
5. `successRate`는 체감 난이도와 크게 어긋나지 않게 설정한다.

## 3) 다국어 룰

1. 기본 문제 본문(`description`, `examples`, `constraints`)은 영어 기준으로 작성한다.
2. 한국어 번역이 필요한 문제는 `problemTranslations`에 `ko` 번들을 추가한다.
3. EN/KO 간 의미 차이(제약, 예시 값, 반환 형식)가 없어야 한다.

## 4) 평가/멘토 호환 룰

1. 테스트 통과 여부는 `JSON.stringify(actual) === JSON.stringify(expected)` 기준임을 염두에 둔다.
2. 문제 설명에 애매한 표현(정렬 여부, 중복 허용 여부 등)을 남기지 않는다.
3. 멘토 피드백이 단계적으로 힌트를 줄 수 있도록 문제의 핵심 패턴이 분명해야 한다.

## 5) UX 룰 (현재 제품 동작 기준)

1. 에디터 하단 테스트 카드에 `Input`과 `Expected Output`이 사전 노출된다.
2. 따라서 예시/테스트케이스 값은 학습 힌트로 활용 가능하되, 정답 로직이 직접 유출되지 않도록 구성한다.
3. 커스텀 테스트 입력 형식(JSON 파싱 기준)과 충돌하지 않게 기본 예시도 JSON 친화적으로 작성한다.

## 6) 출제자 스킬 정의

### Skill A: 문제 모델링
- 자연어 요구사항을 함수 입출력 계약으로 정확히 변환하는 능력
- 애매한 조건을 명시적 규칙으로 바꾸는 능력

### Skill B: 테스트 설계
- 정상/경계/함정 케이스를 균형 있게 만드는 능력
- 오답 패턴(정렬 누락, 중복 처리 누락, 인덱스 오프셋 오류)을 겨냥한 케이스 설계 능력

### Skill C: 난이도 튜닝
- 제약(`n` 크기)에 맞춰 시간복잡도 기대치를 설정하는 능력
- 같은 주제라도 Easy/Medium/Hard로 단계화하는 능력

### Skill D: 설명 작성
- 짧고 오해 없는 설명 작성 능력
- 예시를 통해 핵심 규칙을 빠르게 학습시키는 능력

### Skill E: 로컬라이징
- EN/KO 의미 정합성 유지 능력
- 번역 시 기술 용어 일관성 유지 능력

## 7) 출제 체크리스트

- [ ] `id` 고유성 확인
- [ ] `functionName` = `starterCode` 함수명 일치
- [ ] `examples`와 `testCases` 정합성 확인
- [ ] 경계 케이스 포함 여부 확인
- [ ] 난이도/제약/성공률의 일관성 확인
- [ ] EN/KO 문구 차이 검수
- [ ] 로컬 실행으로 테스트 통과/실패 시나리오 확인
